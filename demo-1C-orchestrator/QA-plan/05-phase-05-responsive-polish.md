# Phase 5 QA: Responsive and polish

**Build file:** `build-plan/phases/05-phase-05-responsive-polish.md`  
**Detail:** `phase-plans/QA/phase-05-qa.md`

## QA goals

Verify ship-ready single-file page across viewports and accessibility.

## Outcomes (check each)

- [ ] No horizontal scroll at 320px from layout bugs
- [ ] Mobile nav usable (open/close, focus trap acceptable)
- [ ] Readable font sizes and touch targets on mobile
- [ ] Contrast sufficient on graphite panels (muted on dark)
- [ ] Skip link works; form and CTAs keyboard accessible
- [ ] All CSS/JS embedded; logos only external images
- [ ] Full page still meets Phase 1–4 acceptance (regression)
- [ ] 60-second comprehension: who, offer, next action

## Pass criteria

All outcomes with evidence.

## Fail conditions

- Broken mobile layout or unusable form on small screens
- Regressed section IDs or CTAs
- Required external stylesheet/script for core experience

**Final visual polish opinions** → `pass_with_risks` only if objective checks pass.
