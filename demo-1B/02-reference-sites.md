# Reference Sites: Vibe Code Academy

This file lists reference sites to study before designing the Vibe Code Academy landing page.

Do not copy these sites directly. Use them as inspiration for structure, messaging, visual hierarchy, trust-building, and conversion flow.

The goal is to create an original landing page that feels practical, clear, credible, and beginner-friendly.

---

## 1. MIT xPRO: Professional Certificate in Advanced Analytics with AI, ML, and Data Science

**URL:**  
https://executive-ed.xpro.mit.edu/professional-certificate-in-advanced-analytics-with-ai-ml-and-data-science

### Summary

This is a professional education landing page for an MIT xPRO certificate program. It is structured around credibility, program outcomes, learner fit, curriculum, faculty, certificate value, and a lead-capture form for downloading a brochure.

The page feels serious, credentialed, and outcome-driven. It is designed for a visitor who is evaluating whether the program is worth their time, money, and professional attention.

### Elements Worth Repeating

- Clear education-program framing near the top of the page.
- Strong hero section with a direct program promise.
- Lead-capture form connected to a concrete asset, such as “Get Your Brochure.”
- Quick facts near the top, such as duration, format, workload, or start date.
- “Who this is for” section that helps visitors self-identify.
- “Key takeaways” section that explains what the learner will be able to do.
- Curriculum or module breakdown to make the learning path feel structured.
- Trust-building sections such as faculty, certificate, program highlights, FAQs, and outcomes.
- Repeated CTA moments throughout a long page.

### How to Apply This to Vibe Code Academy

Use this site as inspiration for credibility and structure. Vibe Code Academy should feel much lighter and more beginner-friendly than MIT xPRO, but it can borrow the idea of clear outcomes, program fit, curriculum structure, and a useful downloadable asset.

## 2. Codecademy

**URL:**  
https://www.codecademy.com/

### Summary

Codecademy is a broad online learning platform focused on helping people build technical skills through interactive, hands-on learning. The homepage emphasizes career growth, practical skills, guided paths, interactive lessons, projects, and learning by doing.

The page is approachable and action-oriented. It makes coding feel accessible by organizing learning into clear subjects, pathways, and practical outcomes.

### Elements Worth Repeating

- Beginner-friendly positioning that makes technical learning feel less intimidating.
- Strong focus on learning by doing, not just watching or reading.
- Clear subject or skill categories that help users understand what they can learn.
- Guided paths that reduce the feeling of “where do I start?”
- Practical outcomes such as building projects, gaining confidence, and developing useful skills.
- Platform screenshots or visual mockups that make the learning experience feel tangible.
- Social proof from real learners.
- Clear signup CTA.

### How to Apply This to Vibe Code Academy

Use this site as inspiration for making AI-assisted building feel accessible. Vibe Code Academy should emphasize hands-on progress, guided steps, simple paths, and the idea that users can build something useful without needing to become expert software engineers first.

## 3. Skillet AI

**URL:**  
https://skillet-ai.com

### Summary

Skillet AI is an automation training and services site focused on helping marketers, business owners, and operators use AI and n8n to automate work. It combines education, tutorials, service offerings, practical use cases, and clear audience-specific examples.

The page is practical, operator-focused, and outcome-oriented. It explains what the user can automate, who the product is for, and why automation matters in business terms.

### Elements Worth Repeating

- Clear promise in the hero section.
- Practical, business-focused language.
- Specific examples of what users can build or automate.
- Audience personas that make the target users feel concrete.
- Use-case cards that show real applications.
- “Do it yourself or have us help” positioning.
- FAQ section that reduces beginner anxiety.
- Clear CTAs such as booking an audit, browsing tutorials, or creating a free account.
- Tutorial/project cards that make the learning content feel real and browsable.

### How to Apply This to Vibe Code Academy

Use this site as inspiration for practical, no-hype positioning. Vibe Code Academy should show concrete things a learner can build, explain who the workshop is for, and make the next step feel simple. It should also borrow the idea of use-case examples and beginner reassurance.

# Design Direction for Vibe Code Academy

Vibe Code Academy should combine the strongest elements of the reference sites:

- MIT xPRO’s credibility and structured program framing.
- Codecademy’s beginner-friendly, hands-on learning orientation.
- Skillet AI’s practical operator language and concrete use-case examples.

The final page should feel:

- Practical
- Friendly
- Clear
- Modern
- Beginner-friendly
- No-hype
- Action-oriented

Avoid making the page feel too academic, too playful, too generic, or too “AI guru.”

---

# Suggested Page Structure

1. Hero section
    - Clear promise
    - Short explanation
    - Primary CTA to get the Prototype Starter Kit

2. Problem section
    - Explain why people get stuck with AI-assisted building

3. Outcome section
    - Show what the learner will be able to create

4. How the workshop works
    - Simple step-by-step explanation

5. Prototype Starter Kit section
    - Explain what the visitor receives after signing up

6. Who this is for
    - Students
    - Marketers
    - Early-stage founders
    - Operators with ideas but limited coding background

7. Example projects
    - Startup landing page
    - Dashboard/report mockup
    - Portfolio site
    - Simple AI-assisted workflow prototype

8. Signup section
    - Email input
    - Clear CTA
    - Success state after submission

9. FAQ / reassurance section
    - Do I need to know how to code?
    - What will I build?
    - What tools will I use?
    - Is this a finished software product?
    - What happens after I sign up?
